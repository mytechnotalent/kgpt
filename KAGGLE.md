# Training KGPT on Kaggle

This guide walks you through pretraining the full GPT-2-small model on Kaggle's free GPU tier. A Kaggle P100 or T4 is roughly **10–15× faster** than Apple Silicon MPS, turning a month-long local run into a few days of Kaggle sessions.

<br>

## Overview

| Item                            | Detail                                    |
| ------------------------------- | ----------------------------------------- |
| **GPU**                         | NVIDIA Tesla P100 16 GB (or T4 16 GB)     |
| **Session limit**               | 12 hours per session                      |
| **Weekly quota**                | 30 hours of GPU time                      |
| **Estimated speed**             | ~3–4 s/iter (vs ~43 s/iter on M2 Max MPS) |
| **Total time for 50 000 iters** | ~42–56 hours (~4–5 sessions over 2 weeks) |

> **Note:** The free GPU quota resets every Saturday at 00:00 UTC.

<br>

## Prerequisites

- A [Kaggle account](https://www.kaggle.com/) (free)
- The `data/train.bin` (17 GB) and `data/val.bin` (86 MB) files generated by `prepare_data.py` on your local machine
- This repository pushed to GitHub (already done)

<br>

## Step 1 — Upload Training Data as a Kaggle Dataset

Kaggle allows up to **20 GB per dataset**, so `train.bin` + `val.bin` (~17.1 GB total) fits.

1. Go to [kaggle.com/datasets](https://www.kaggle.com/datasets) and click **+ New Dataset**.
2. Name it `kgpt-openwebtext-tokens` (or any name you prefer).
3. Upload both files:
   - `data/train.bin` (17 GB — this will take a while on most connections)
   - `data/val.bin` (86 MB)
4. Set visibility to **Private** (unless you want to share it).
5. Click **Create** and wait for the upload to finish.

> **Tip:** If your upload keeps timing out, you can use the [Kaggle API](https://github.com/Kaggle/kaggle-api):
> ```bash
> pip install kaggle
> kaggle datasets init -p data/
> # Edit data/dataset-metadata.json with your dataset name
> kaggle datasets create -p data/
> ```

<br>

## Step 2 — Create a Kaggle Notebook

You **do** use a Kaggle Notebook (`.ipynb`), but only as a lightweight runner. All the real code lives in your `.py` files — the notebook just clones the repo, links the data, and executes the training script.

1. Go to [kaggle.com/code](https://www.kaggle.com/code) and click **+ New Notebook**.
2. In the right sidebar:
   - **Accelerator** → select **GPU P100** (or **GPU T4 x2** if available).
   - **Persistence** → set to **Variables and Files** (so outputs survive kernel restarts within a session).
3. Under **Data** → **+ Add Data**, search for your `kgpt-openwebtext-tokens` dataset and add it.

The dataset will be mounted at:
```
/kaggle/input/datasets/mytechnotalent/kgpt-openwebtext-tokens/
```

<br>

## Step 3 — Notebook Cells

Paste these cells into your Kaggle Notebook:

### Cell 1 — Install dependencies and clone repo
```python
!pip install -q tiktoken

import os
os.chdir("/kaggle/working")

# Clone the repo (only if not already cloned from a previous session)
if not os.path.exists("kgpt"):
    !git clone https://github.com/mytechnotalent/kgpt.git

os.chdir("kgpt")
!git pull  # grab latest changes
```

### Cell 2 — Symlink the data
```python
import os

os.makedirs("data", exist_ok=True)

# Path to your uploaded Kaggle dataset
INPUT = "/kaggle/input/datasets/mytechnotalent/kgpt-openwebtext-tokens"

for fname in ["train.bin", "val.bin"]:
    src = os.path.join(INPUT, fname)
    dst = os.path.join("data", fname)
    if not os.path.exists(dst):
        os.symlink(src, dst)
        print(f"Linked {src} → {dst}")
    else:
        print(f"Already exists: {dst}")
```

### Cell 3 — Train
```python
!python train.py
```

That's it. The training loop will:
1. Detect the CUDA GPU automatically.
2. Print loss every 2 000 iterations.
3. Save checkpoints to `kgpt_checkpoint.pt` every `eval_interval` steps.
4. When the session ends (or you interrupt), resume from the latest checkpoint next time.

### Cell 4 — (After training) Download the model
```python
# Copy final weights to Kaggle output so you can download them
import shutil
for f in ["kgpt_pretrained.pt", "kgpt_checkpoint.pt"]:
    if os.path.exists(f):
        shutil.copy(f, f"/kaggle/working/{f}")
        print(f"Copied {f} to /kaggle/working/")
```

Then click **Save Version** → **Save & Run All** → download the output files from the **Output** tab.

<br>

## Step 4 — Resume Across Sessions

`train.py` saves a checkpoint (`kgpt_checkpoint.pt`) every `eval_interval` iterations containing the model weights, optimizer state, and current iteration number.

**To resume in a new Kaggle session:**

1. Before your session expires, the checkpoint is automatically saved.
2. In your next session, upload `kgpt_checkpoint.pt` as a **new Kaggle Dataset** (or add it to your existing one):
   ```
   /kaggle/input/datasets/mytechnotalent/kgpt-openwebtext-tokens/kgpt_checkpoint.pt
   ```
3. Add an extra cell before training to copy it in:
   ```python
   import shutil, os
   ckpt_src = "/kaggle/input/datasets/mytechnotalent/kgpt-openwebtext-tokens/kgpt_checkpoint.pt"
   ckpt_dst = "/kaggle/working/kgpt/kgpt_checkpoint.pt"
   if os.path.exists(ckpt_src) and not os.path.exists(ckpt_dst):
       shutil.copy(ckpt_src, ckpt_dst)
       print("Restored checkpoint")
   ```
4. Run `!python train.py` — it will automatically detect and resume from the checkpoint.

## Kaggle Tips

| Tip                                      | Detail                                                                  |
| ---------------------------------------- | ----------------------------------------------------------------------- |
| **Monitor GPU usage**                    | Add `!nvidia-smi` in a cell to check memory and utilization             |
| **Keep sessions alive**                  | Kaggle may disconnect idle sessions — keep the browser tab open         |
| **Save early**                           | Click **Save Version** before the 12-hour limit to preserve outputs     |
| **Track progress**                       | Loss at step 0 should be ~10.97 (random); aim for < 3.5 after 50K iters |
| **Weekly rhythm**                        | ~10 hrs/week of training → 50K iters in ~5–6 weeks at 30 hrs/week quota |
| **Alternative: Kaggle competitions GPU** | Some competitions grant extra GPU hours                                 |
